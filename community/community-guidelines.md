# TensorMarketData Community Guidelines & Rules

## 1. Our Core Values

These principles guide everything we do at TensorMarketData:

### 1.1 Developer-First
We exist to help developers build amazing things. Every decision we make starts with the question: "Does this help developers?"

### 1.2 Transparency
No hidden fees. No bait-and-switch. No fine print. What you see is what you get.

### 1.3 Quality
We verify every dataset. Every provider. Every API. Quality is non-negotiable.

### 1.4 Community
You make this platform valuable. Your feedback shapes our roadmap. Your success is our success.

## 2. Code of Conduct

### 2.1 Expected Behavior

We expect all community members to:

**Be Respectful**
- Treat everyone with respect and dignity
- Disagree professionally, without personal attacks
- Consider different perspectives and experiences

**Be Helpful**
- Answer questions when you can
- Point people to resources and documentation
- Share knowledge generously

**Be Honest**
- Don't misrepresent your products or services
- Give accurate information
- Acknowledge when you don't know something

**Be Constructive**
- Provide feedback that helps others improve
- Celebrate community wins
- Support newcomers and beginners

### 2.2 Unacceptable Behavior

The following will result in immediate action:

**Harassment & Discrimination**
- Hate speech or symbols
- Personal attacks or insults
- Doxxing or threats
- Unwanted sexual attention

**Spam & Self-Promotion**
- Repeated posting of promotional content
- Unapproved advertisements
- Affiliate link spam
- Excessive self-promotion without contributing

**Disruption**
- Intentional provocation
- Bad-faith arguments
- Sabotaging community resources
- Impersonating team members or other users

**Illegal Activity**
- Sharing pirated or stolen data
- Discussing illegal activities
- Attempting to bypass security measures
- Fraudulent transactions or reviews

## 3. Platform-Specific Rules

### 3.1 Discord Server Rules

**General Channels (#general, #introductions)**
- Stay on topic (AI agents, data, development)
- No excessive self-promotion (max 1 post per week without approval)
- Keep discussions respectful and constructive
- Use threads for long conversations

**Support Channels (#api-help, #bug-reports)**
- Search before asking (your question may already be answered)
- Be specific: include code snippets, error messages, what you've tried
- One issue per thread (start new thread for new issues)
- Be patient — volunteers answer when they can

**Bug Reports**
- Search existing reports first
- Include: steps to reproduce, expected vs actual behavior, screenshots
- Tag appropriately (severity, category)
- Don't bump reports — the team sees them all

### 3.2 GitHub Repository Rules

**Issues**
- Search before creating (avoid duplicates)
- Use issue templates (bug report, feature request)
- Provide all requested information
- Be responsive to maintainer questions

**Pull Requests**
- Follow coding style guidelines
- Include tests for new features
- Update documentation
- Keep PRs focused and reviewable
- Be responsive to review feedback

**Discussions**
- Stay on topic (TensorMarketData product, SDK, API)
- Be helpful to other developers
- No spam or self-promotion

### 3.3 Social Media Rules

**@Hexy_Hexborne / Twitter**
- Engage respectfully with all comments
- Correct misinformation politely
- Don't feed trolls (block obvious spam/bad actors)
- Stay professional while being personable

**LinkedIn**
- Professional tone
- No aggressive self-promotion
- Share valuable content, not just product pitches

### 3.4 Newsletter & Email Guidelines

**Community Emails**
- Respect unsubscribe requests immediately
- No misleading subject lines
- Keep content valuable, not just promotional
- Allow easy unsubscription

## 4. Data Provider Guidelines

### 4.1 Quality Standards

All data providers must maintain:

**Accuracy**
- Data must match real-world values (verified spot-check)
- Errors must be corrected within 24 hours
- Known issues must be documented

**Freshness**
- Update frequencies must match documentation
- Stale data must be clearly labeled
- Delays must be communicated

**Completeness**
- Fields must be populated as documented
- Missing data must be marked null, not omitted
- Schema changes must be announced 7 days in advance

**Reliability**
- 99%+ uptime required
- Response times must match SLAs
- Rate limits must be enforced and documented

### 4.2 Provider Responsibilities

**Listing Accuracy**
- Describe data accurately (no overpromising)
- Disclose limitations upfront
- Show sample data before purchase
- Update descriptions when data changes

**Customer Support**
- Respond to inquiries within 48 hours
- Address bugs and issues promptly
- Communicate about outages proactively

**Compliance**
- Have legal rights to distribute data
- Respect data source terms of service
- Don't infringe on copyrights
- Protect user data per privacy policy

### 4.3 Provider Prohibited Actions

- Fake reviews or ratings
- Misleading descriptions
- Selling data you don't have rights to
- Overstating quality or freshness
- Hiding fees or limitations
- Gaming the marketplace rankings
- Attempting to extract user information

### 4.4 Provider Consequences

| Violation | Consequence |
|-----------|-------------|
| Minor (misleading description) | Warning + 48h to fix |
| Moderate (quality issues) | Temporary delisting + 7 days to fix |
| Moderate (slow support) | Warning + 14 days to improve |
| Severe (fake reviews) | Permanent delisting |
| Severe (illegal data) | Immediate ban + legal action |
| Repeat offenses | Escalated consequences |

## 5. Data Consumer Guidelines

### 5.1 Usage Rights

When you purchase data from TensorMarketData:

**You May:**
- Use data in your AI agents and applications
- Cache data for performance (within rate limits)
- Share within your organization
- Build derivatives and insights

**You May NOT:**
- Resell data without provider permission
- Scrape marketplace instead of using APIs
- Share API keys publicly
- Attempt to reverse-engineer data sources
- Use data for illegal purposes

### 5.2 Ethical Use

We expect ethical data usage:

**Good:**
- Using financial data for legitimate research
- Aggregating weather data for logistics
- Analyzing news sentiment for insights
- Building helpful AI applications

**Bad:**
- Market manipulation with insider data
- Surveillance or tracking without consent
- Discrimination in hiring/insurance/credit
- Any use violating provider terms

### 5.3 Feedback & Reporting

**If you find issues:**
- Report quality problems in #bug-reports or Discord
- Rate providers honestly
- Document evidence (screenshots, timestamps)
- Give providers a chance to fix before public complaints

**If you're mistreated:**
- Contact moderators privately first
- Document the incident
- Allow us to investigate and respond
- Escalate if resolution is unsatisfactory

## 6. Content Moderation

### 6.1 Moderation Principles

1. **Transparency** — Explain decisions when possible
2. **Proportionality** — Response fits the offense
3. **Consistency** — Apply rules fairly to all
4. **Recovery** — Offer path back after violations

### 6.2 Moderation Process

**Step 1: Detection**
- Automated flagging (spam, keywords)
- User reports
- Moderator巡查

**Step 2: Assessment**
- Review evidence
- Check for patterns
- Consult team if unclear

**Step 3: Action**
- Minor: Warning, remove content
- Moderate: Temporary mute/ban
- Severe: Permanent ban, remove content

**Step 4: Communication**
- Notify affected user
- Explain reason for action
- Offer appeal process

### 6.3 Appeal Process

If you believe a moderation decision was wrong:

1. Email moderations@tensormarketdata.com
2. Include: username, action taken, reason you believe it's wrong
3. We'll review within 72 hours
4. Decision is final after appeal
